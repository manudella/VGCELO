How the ratings & stats work

Everything on this site is computed deterministically from public RK9 data. This page documents exactly how, so the numbers are auditable.

What counts

  • Source: RK9 — the official tournament data (placings, team lists, and round-by-round pairings and results).
  • Events: majors only — Regional Championships and above (Regionals, Special Events, Internationals, Worlds). Cups and locals are excluded.
  • Window: from the {{ config.scrape.first_season }} season onward, when the Scarlet/Violet data format stabilised.
  • Division: Masters only.
  • A match is one best-of-three set between two players in a round. Byes are recorded but are not rated.

The Elo rating

Every player starts at {{ config.elo.initial_rating|int }}. After each match the winner takes points from the loser. The amount depends on how surprising the result was: beating a much higher-rated opponent earns a lot; beating a much lower-rated one earns little.

The expected score for player A against player B is the standard logistic curve:

EA = 1 / (1 + 10(RB − RA) / 400)

and the update is R′ = R + K · (S − E), where S is 1 for a win, 0 for a loss (½ for a tie).

The K-factor

K controls how much a single result moves your rating. It starts high so an unknown player converges on their true rating quickly, then decreases linearly to a stable value over their first {{ config.elo.k_decay_games }} rated games:

K(g) = {{ config.elo.k_initial }} − ({{ config.elo.k_initial }} − {{ config.elo.k_final }}) · (g / {{ config.elo.k_decay_games }}), capped at {{ config.elo.k_final }}

where g is how many rated games the player had before the match. So a brand-new player uses K = {{ config.elo.k_initial }}; after {{ config.elo.k_decay_games }}+ games it settles at {{ config.elo.k_final }}. There is no tournament-tier weighting and no seasonal reset — every game counts the same and ratings are continuous.

Ratings are recomputed from scratch each update by replaying every match in chronological order, so results are fully reproducible.

Glicko-2 & GXE (Showdown-style)

Alongside Elo we compute Glicko-2, the system Pokémon Showdown uses. As well as a rating it tracks a rating deviation (RD — how confident we are) and a volatility. Each tournament is treated as one Glicko-2 “rating period”: all of a player's matches at an event update their rating together, and an inactive player's RD slowly grows (we become less sure of a rusty player).

GXE (“X-Act estimate”) is derived from a player's Glicko rating and RD exactly as Showdown defines it: the estimated percentage chance they would beat an average ladder player. A player on a long win streak against strong opposition trends toward a high GXE with a low (tight) RD.

Starting values: rating {{ config.raw.glicko.initial_rating|int }}, RD {{ config.raw.glicko.initial_rd|int }}, volatility {{ config.raw.glicko.initial_vol }}, system constant τ = {{ config.raw.glicko.tau }}.

Player metrics

  • Peak rating — the highest rating ever reached.
  • Win rate — rated wins ÷ (wins + losses); byes and ties are excluded from the denominator.
  • Longest win / loss streak — the longest run of consecutive rated wins (or losses) across the player's whole career, in chronological order.
  • Biggest upmatch — the single rated win that earned the most Elo, i.e. the biggest upset against a higher-rated opponent.
  • Per-Pokémon win rate — the player's match record in the events where that Pokémon was on their team.
  • Signature Pokémon — the species they have brought to the most majors.

Pokémon metrics

  • Usage % — the share of all published team lists that include the Pokémon.
  • Win rate — aggregated match win rate of every player in the events where they used it.
  • Set breakdown — among all copies of the Pokémon, the percentage running each item, Tera type, ability and move. (This is what powers lines like “% of Flutter Mane holding Choice Specs”.) Moves are a percentage of copies that ran the move; the four bars therefore sum to roughly 400%.

Caveats

  • VGC sets are best-of-three; the source reports the set winner, not individual games, so Elo is per-set.
  • Team lists are only available where the source published them (newer events have them; some older events carry results only), so usage stats lean toward more recent seasons.
  • Set data depends on the era: Scarlet/Violet lists expose Tera types, while the post-SV Regulation M-A (Champions) era exposes stat alignment instead. EV spreads are not provided in either, so they aren't shown. Pokémon stats can be filtered by regulation.
  • Players are keyed by name; rare name collisions could merge two people.
